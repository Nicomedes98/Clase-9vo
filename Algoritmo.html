Primero vemos ejemplos, después copiamos definiciones y practicamos

📝Problema→ 🔢Pasos comunes→ 🧾Pseudocódigo→ ✅Revisión

Primero miramos un ejemplo

La idea es entender qué pasa antes de copiar definiciones. Un algoritmo puede escribirse con pasos comunes y después pasarse a pseudocódigo.

Ejemplo 1: saludar a una persona

Problema: queremos que el algoritmo pida un nombre y muestre un saludo.

  1. Pedir el nombre de la persona.
  2. La persona escribe su nombre.
  3. El algoritmo muestra un saludo usando ese nombre.
📥Entrada

El dato que entra es el nombre que escribe la persona.

⚙️Proceso

El algoritmo usa ese dato para armar un saludo.

📤Salida

El resultado es el mensaje que aparece en pantalla.

Algoritmo Saludo
    Escribir "Ingrese su nombre"
    Leer nombre
    Escribir "Hola ", nombre
FinAlgoritmo
Algoritmo Saludo
Indica el inicio y el nombre del algoritmo.
Escribir
Muestra un mensaje en pantalla. En este caso, pide el nombre.
Leer nombre
Permite que la persona ingrese un dato. Ese dato queda identificado como nombre para usarlo después.
Escribir "Hola ", nombre
Muestra la salida: un saludo usando el dato ingresado.
FinAlgoritmo
Indica que el algoritmo terminó.

Ejemplo 2: sumar dos números

Problema: queremos pedir dos números, sumarlos y mostrar el resultado.

  1. Pedir el primer número.
  2. Pedir el segundo número.
  3. Sumar los dos números.
  4. Mostrar el resultado.
📥Entrada

Los datos que entran son el primer número y el segundo número.

⚙️Proceso

El algoritmo realiza la suma de los dos números.

📤Salida

El resultado que se muestra es la suma.

Algoritmo Suma
    Escribir "Ingrese el primer número"
    Leer numero1
    Escribir "Ingrese el segundo número"
    Leer numero2
    resultado = numero1 + numero2
    Escribir "El resultado es: ", resultado
FinAlgoritmo
Escribir
Muestra una indicación para que la persona sepa qué dato debe ingresar.
Leer numero1 y Leer numero2
Permiten ingresar los dos datos que necesita el algoritmo.
resultado = numero1 + numero2
Es el proceso: se realiza la suma.
Escribir "El resultado es: ", resultado
Muestra la salida final del algoritmo.
A tener en cuenta: por ahora usamos nombres como numero1, numero2 y resultado solo para identificar los datos. Más adelante se trabajará el concepto de variable con mayor detalle.

🧉 Ejemplo 3: preparar mate usando Leer y Escribir

Consigna: copiá el algoritmo en el cuaderno y transformalo a pseudocódigo. Esta vez no alcanza con mostrar pasos: el algoritmo también debe pedir datos y usarlos en una salida.

La idea es practicar la diferencia entre Escribir —mostrar un mensaje— y Leer —recibir un dato que escribe la persona—.

🧾 Algoritmo en pasos comunes

  1. Pedir el nombre de la persona que va a preparar el mate.
  2. Pedir cuántas personas van a compartir.
  3. Mostrar un mensaje inicial con esos datos.
  4. Calentar el agua.
  5. Colocar yerba en el mate.
  6. Colocar la bombilla.
  7. Cebar con agua caliente.
  8. Mostrar el mensaje final: “mate listo”.
📥 Entrada→⚙️ Proceso→📤 Salida

✍️ Para completar en el cuaderno

Algoritmo PrepararMate Escribir "Ingrese su nombre" Leer ____________ Escribir "Ingrese cuántas personas comparten" Leer ____________ Escribir "Vamos a preparar mate para ", ____________ Escribir "Cantidad de personas: ", ____________ Escribir "Calentar el agua" Escribir "Colocar yerba en el mate" Escribir "Colocar la bombilla" Escribir "Cebar con agua caliente" Escribir "Mate listo para compartir" FinAlgoritmo
Para pensar antes de escribir: las líneas con Leer reciben datos. Las líneas con Escribir muestran mensajes o resultados. En los espacios en blanco deben repetir el mismo nombre del dato que se leyó, por ejemplo: nombre o cantidad.

    • Salida: mostrar nombre y grupo.

    🧮Algoritmo 2: Doble de un número

    Crear un algoritmo que pida un número, calcule el doble y muestre el resultado.

    • Entrada: un número.
    • Proceso: multiplicar ese número por 2.
    • Salida: mostrar el doble.
